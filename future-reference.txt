🚀 Day 1: Setup & Environment
Goal: Get PostgreSQL and MongoDB running via Docker, set up our Node.js skeleton, and successfully connect to both databases.

Step 1: Spin up the Databases (Docker)
Open your terminal and run these two commands. This will pull the images (if you don't have them) and start the containers in the background (-d).

PostgreSQL:

Bash
docker run -d --name postgres-lit -e POSTGRES_PASSWORD=secret -p 5432:5432 postgres:16
MongoDB:

Bash
docker run -d --name mongo-lit -p 27017:27017 mongo:7
Step 2: Initialize the Node.js Project
Create your project folder, initialize npm, and install the required packages.

Bash
mkdir litcritic
cd litcritic
npm init -y
npm i express pg mongodb dotenv
npm i -D nodemon
Next, open your package.json and add a start script for nodemon so the server auto-restarts when we change code:

JSON
"scripts": {
  "dev": "nodemon index.js"
}
Step 3: Environment Variables
Create a file named .env in the root of your litcritic folder. This stores our connection strings.

Code snippet
PORT=3000
PG_URI=postgres://postgres:secret@localhost:5432/postgres
MONGO_URI=mongodb://localhost:27017/litcritic
Step 4: The Code (index.js)
Create an index.js file. We will import our drivers, connect to both databases, and expose a /ping route.

JavaScript
require('dotenv').config();
const express = require('express');
const { Client } = require('pg');
const { MongoClient } = require('mongodb');

const app = express();
const port = process.env.PORT || 3000;

// Initialize Database Clients
const pgClient = new Client({ connectionString: process.env.PG_URI });
const mongoClient = new MongoClient(process.env.MONGO_URI);

async function startServer() {
  try {
    // 1. Connect to PostgreSQL
    await pgClient.connect();
    console.log('✅ PostgreSQL connected successfully!');

    // 2. Connect to MongoDB
    await mongoClient.connect();
    console.log('✅ MongoDB connected successfully!');
    
    // We can explicitly create/select the database in Mongo
    const mongoDb = mongoClient.db('litcritic');

    // 3. Simple Ping Route
    app.get('/ping', (req, res) => {
      res.status(200).json({ 
        success: true, 
        message: "Postgres & MongoDB connected! 🚀" 
      });
    });

    // 4. Start Express Server
    app.listen(port, () => {
      console.log(`📚 LitCritic server running at http://localhost:${port}`);
    });

  } catch (error) {
    console.error('❌ Database connection failed:', error);
    process.exit(1);
  }
}

startServer();
Step 5: Run It!
Back in your terminal, run:

Bash
npm run dev
You should see logs confirming both databases connected and the server is running. Open your browser or Postman and go to http://localhost:3000/ping.

📝 Day 1 Concept Checklist:
Connection Strings: Notice how Postgres uses postgres://user:pass@host:port/db while Mongo uses mongodb://host:port/db.

Database creation: In Postgres, the database postgres exists by default. In MongoDB, the litcritic database doesn't actually get created on disk until we insert our first document into it (which we'll do later!).

Async/Await: Database connections are I/O operations, meaning they take time. We use async/await to ensure our server only starts after the databases are ready.
